# 米子市政の見える化サイト - VISION

## ゴール
米子市民が、市政に関する情報に簡単にアクセスでき、
理解しやすい形で情報に触れられるようにする。

## 市公式サイトとの差別化
米子市公式サイトは「データの置き場」であり、市民は自分で必要な情報を
探さなければならない。本サイトは「情報の見せ方」で価値を作る。

データを増やすだけでは市公式サイトのコピーになり、価値がない。
**統合・整理・分かりやすさ**で勝負する。

## 設計の3つの軸

### 1. 人軸
議員・市長等の活動を統合して見せる。
散らばった情報(基本情報・質問・議決・SNS等)を1つのプロフィールに集約。

### 2. 時間軸
いつ何が起きたかを追える。
新着タイムラインを核に、議会・行政・市政の動きを時系列で並べる。

### 3. テーマ軸(将来)
市民の関心事(子育て・福祉・防災等)ごとに情報を束ねる。
市公式は縦割り、本サイトは市民の視点での横串。

## やらないこと

- 市公式サイトの単純コピー
- データを増やすだけのタブ追加
- 専門用語を多用した難しい解説
- 全部の情報を載せること(価値の低い情報は載せない判断をする)
- 議員個人の連絡先(電話・メール)・自宅住所・生年月日等の個人情報は、公式サイトに掲載されていても収集・掲載しない

## やること

- 散らばった情報の統合
- 時系列での提示
- 検索・フィルタ・絞り込み
- 数値の可視化(グラフ・チャート)
- 分かりやすい言葉での説明
- 自動更新による情報の鮮度維持

## サイト構造の目標(改訂)

最終形は「タブ羅列」ではなく、3層構造:

1. **ホーム**: 最新の動き、注目情報、横断検索
2. **ハブページ**:
   - 議員ハブ(議員一覧 + 個別プロフィール)
   - 議会活動ハブ(質問・議決・日程)
   - 市の数字ハブ(統計・指標)
   - お金ハブ(予算・決算・入札等)
3. **個別ビュー**: ハブから掘り下げた詳細

現在は「タブ羅列」段階だが、Phase 4 で「ハブ構造」に再編する。

鳥取県ページでは、鳥取県内19市町村(4市+15町村、14町1村)を地図上に示し、対応済み/未対応の導線を分ける。

## データ追加の判断基準

新しいデータソースを追加する前に、以下を満たすか確認する:

- [ ] このデータは「人軸」「時間軸」のいずれかに統合できるか?
- [ ] 市公式サイトより分かりやすく見せられるか?
- [ ] 追加することで全体の見通しが悪くならないか?
- [ ] 5年後も価値を持つデータか?
- [ ] 自動更新できるか?(運用負担を増やさないか?)

これらに2つ以上「No」なら、追加を見送る。

## 評価基準

このサイトの成功は、以下で測る:

- **3分テスト**: 友人・家族にサイトを紹介して、3分以内で
  「米子市の○○について知りたい」を解決できるか?
- **継続テスト**: 月1回以上、誰かが定期的に見に来てくれるか?
- **市民の声テスト**: 「市公式より見やすい」というフィードバックが得られるか?

## 開発スタイル

- vanilla JS(フレームワーク不使用)
- 機能別にファイルを分割し、ES Modules で連携
- データは JSON ファイル(リポジトリ管理)
- 自動データと手動データを分離
- 段階的進行(Phase ごとに動くものを完成させてから次へ)
- 最小変更スタイル(過剰な機能追加を避ける)
- ビルド工程なし(直接ブラウザで実行可能を維持)

## 技術スタック

- スクレイピング: Python + requests + BeautifulSoup
- フロント: HTML + vanilla JS + CSS
- データ: JSON ファイル
- 公開: GitHub Pages
- 自動更新: GitHub Actions
- PDF処理(Phase 3): pdfminer / pypdf / pdfplumber 等を検討
- AI活用(Phase 5): Claude Haiku 等の軽量モデルを検討

## 参考にする先行事例

### 国会議員マップ
- URL: https://kokkaimap.jp/
- 国政レベル(衆参712名)で同様のことを実現している個人プロジェクト
- AI要約に Anthropic 社の Claude Haiku 4.5 を使用
- 一次ソース由来の発言・投票記録 + AI要約 + 口コミの統合
- 4つの入り口(郵便番号・テーマ・ニュース・診断)で多様な動機に対応

参考にする要素:
- 個別議員ページの構成(発言・投票・AI要約・口コミ)
- テーマ別ビュー(税金・子育て等から入る)
- やさしくニュース解説(専門用語を噛み砕く)
- AI要約の実装(Claude Haiku 4.5)
- 免責事項の書き方

米子プロジェクトの独自性:
- 市議会レベルの身近さ(国会と違って身近)
- 委員会活動の詳細(市議会は委員会が中心)
- 市政の数字(数字で知る米子市)
- 新着タイムライン(複数ソース統合)
- 市民意見募集との連携
