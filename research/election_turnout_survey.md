# 選挙投票率データ取得可能性調査

調査日: 2026-06-12

対象: 鳥取県知事選、鳥取県議選、鳥取市・米子市・倉吉市・境港市の市長選/市議選の投票率推移。

## 3行要約

- 県知事選・県議選は、鳥取県選管の「選挙の記録(投票率など)」と総務省の地方選挙結果調/SSDSで時系列化できる。県単位だけなら e-Stat SSDS `0000010107` に投票率項目がある。
- 4市の市長選/市議選は自治体サイトごとに形式がばらつく。鳥取市と境港市は過去分の一覧性が高く、米子市は直近のHTML表が取りやすいが過去分の発見性が弱い。倉吉市は確認範囲では市議選の無投票結果が中心。
- 全国展開は、SSDSで県レベルを自動取得し、市区町村選挙は「自治体ごとの年1手動更新+軽量パーサ」が現実的。単一APIで全国市区町村の市長/市議投票率を安定取得するルートは確認できない。

## 総務省・e-Stat/SSDS

### 総務省

- 選挙関連資料トップ: `https://www.soumu.go.jp/senkyo/senkyo_s/data/index.html`
- 地方選挙結果調: `https://www.soumu.go.jp/senkyo/senkyo_s/data/chihou/index.html`
- 地方選挙結果一覧: `https://www.soumu.go.jp/senkyo/senkyo_s/data/chihou/ichiran.html`
- 都道府県知事選挙結果: `https://www.soumu.go.jp/senkyo/senkyo_s/data/chiji/index.html`
- 指定都市市長選挙結果: `https://www.soumu.go.jp/senkyo/senkyo_s/data/shityou/index.html`

地方選挙結果一覧では、第15回(平成15年4月)から第19回(平成31年4月)の統一地方選挙結果PDFを確認した。第20回(令和5年4月)は総務省ページでは今回の確認範囲で一覧に見えず、鳥取県選管の第20回統一地方選挙ページを正にする方が安定する。

都道府県知事選挙結果ページは平成22年以降の各都道府県知事選PDFが並ぶ。ただし鳥取県知事選は統一地方選挙と同時期のため、単独知事選PDF一覧より地方選挙結果調/県選管ページに寄せるのが安全。

指定都市市長選挙結果は平成23年以降の政令市のみ。鳥取市・米子市・倉吉市・境港市の市長選には使えない。

### e-Stat/SSDS

e-Stat API `getStatsList` を `選挙`、`投票率`、`地方選挙` で確認したところ、投票率として実用的に該当したのは社会・人口統計体系の都道府県表だった。

- `statsDataId=0000010107` 社会・人口統計体系 G 文化・スポーツ
  - `G6301_衆議院議員選挙投票率（小選挙区）`
  - `G6302_衆議院議員選挙投票率（比例代表）`
  - `G6303_参議院議員選挙投票率（比例代表）`
  - `G6304_参議院議員選挙投票率（選挙区）`
  - `G6305_都道府県議会議員選挙投票率`
  - `G6306_都道府県知事選挙投票率`
  - `G6307_市区町村議会議員選挙投票率`
  - `G6308_市区町村長選挙投票率`
- `statsDataId=0000020107` 市区町村表は確認した範囲で選挙/投票率項目なし。

したがって、SSDSは「鳥取県としての県知事/県議/市区町村長/市区町村議会投票率の県内集計」には使えるが、鳥取市・米子市・倉吉市・境港市それぞれの市長選/市議選投票率には使えない。

`選挙` 検索では地方財政状況調査の「選挙費の内訳」もヒットしたが、投票率ではないため本用途から除外する。

## 全国展開時の再現性

| レイヤー | 再現性 | 理由 |
| --- | --- | --- |
| e-Stat/SSDS 都道府県単位 | 高 | `0000010107` に投票率項目があり、県別の長期推移をAPI取得できる見込み |
| 総務省 地方選挙結果調 | 中 | 統一地方選・知事選の全国資料として有用だがPDF中心。自治体別の市長/市議をそのままAPI取得する用途には弱い |
| 都道府県選管 | 中 | 鳥取県のように「選挙の記録」が整理されている県は使えるが、PDF分割やページ構成は県ごとに異なる |
| 市区町村選管 | 低-中 | HTML表、PDF、Excel、無投票だけの簡易ページ、最新回のみ掲載などばらつきが大きい |

全国展開では、まず SSDS で県単位投票率を押さえ、自治体ページは `source_registry` でURL・形式・選挙種別・抽出方式を個別登録する設計が必要。自治体サイトを完全自動探索するより、年1回の手動確認でURL一覧を更新し、既知形式だけ軽量パースする方が安定する。

## 推奨ルート

1. 県知事選/県議選の県単位推移は、SSDS `0000010107` を第一候補にする。県選管PDFは検算・詳細確認用に使う。
2. 鳥取市は「選挙の結果」一覧を入口にし、ExcelがあるものはExcel、PDFのみは手動確認またはPDFテキスト抽出で年1更新する。
3. 米子市は直近HTML表パーサを作れば取得できるが、過去URL一覧は手動で補完する。
4. 境港市は過去結果一覧からHTML表を辿る自動取得候補にする。
5. 倉吉市は無投票・ページ不足のため、当面は手動年1更新または見送り。表示する場合は無投票を明示する。

総合判定: 初期実装では「県単位はSSDS自動」「市区町村は手動年1更新ファイル」を推奨。市区町村の完全自動取得は、境港市・米子市のようなHTML表が安定している自治体から段階導入する。

## 落とし穴

- 無投票は投票率が存在しない。`0%` ではなく `null` とし、`uncontested=true` を持つ。
- 市議選は補欠選挙と一般選挙が混じる。推移に入れるのは原則として一般選挙のみ。
- 県議選は選挙区ごとに無投票/投票ありの差がある。県全体投票率と選挙区別投票率を混ぜない。
- 鳥取市は平成16年の市町村合併後の境界で見るのが無難。2004年前後をまたぐ市単位比較は対象区域が変わる。
- 総務省の地方選挙結果調は全国比較の基礎資料として強いが、PDF中心で、4市の個別時系列を単一APIで取得する用途には向かない。
- e-Stat検索で「選挙費」など投票率ではない統計がヒットする。項目名まで確認して除外する。
- 自治体サイトはリニューアル時にURLが大きく変わる。鳥取市のように旧ページが新URLへ再構成されるため、URLだけでなくページID/タイトルも保持する。
